# Requirements Document

## Introduction

This document covers the major overhaul of the Auto News Station app — a Kenyan entertainment and news social media automation platform deployed at https://auto-news-station-roylandz-media.vercel.app/dashboard. The platform auto-scrapes video content from 50+ sources, generates AI captions and branded thumbnails, and posts Reels/videos to Instagram and Facebook via the Meta Graph API.

The overhaul addresses three critical bugs (SSE parsing, dark IG thumbnails, no post-success navigation), then delivers a video-first cockpit UI, a dedicated Compose page, expanded autonomous video scraping from football and entertainment platforms, and a comprehensive set of Composer and platform-wide improvements.

## Glossary

- **Composer**: The dedicated page at `/composer` (currently exists) that handles manual video URL input, preview, caption editing, and posting to IG + FB.
- **Cockpit**: The live operations dashboard showing post history, stats, and autonomous pipeline controls.
- **SSE**: Server-Sent Events — the streaming protocol used by `/api/post-video` to emit progress updates to the client.
- **R2**: Cloudflare R2 object storage used to stage video and image files before Meta Graph API ingestion.
- **IG**: Instagram — posts are published as Reels via the Meta Graph API.
- **FB**: Facebook — posts are published as videos via the Meta Graph API.
- **Meta_Graph_API**: The Facebook/Instagram Graph API v19.0 used for all social publishing.
- **Video_Pipeline**: The end-to-end flow: scrape → resolve URL → download → stage in R2 → generate thumbnail → post to IG + FB → log.
- **Cover_Image**: A branded thumbnail image (1080×1350 px) generated by `image-gen.ts` and passed as `cover_url` to the IG Reels API to replace the default black frame.
- **Worker**: The Cloudflare Worker at `auto-ppp-tv.euginemicah.workers.dev` that handles KV storage, R2 staging, cron triggers, and post logging.
- **VideoItem**: The TypeScript interface in `video-sources.ts` representing a scraped video with `id`, `title`, `url`, `directVideoUrl`, `thumbnail`, `sourceName`, `sourceType`, and `category`.
- **Bloom_Filter**: An in-memory probabilistic dedup structure used in `video-sources.ts` to avoid re-fetching already-seen video IDs within a cold-start session.
- **Composer_Page**: The `/composer` route (`src/app/composer/page.tsx`) containing the Cockpit, Compose, Sources, and History tabs.
- **Dashboard_Page**: The `/dashboard` route (`src/app/dashboard/page.tsx`) — the main command center with article-post history and pipeline controls.
- **EARS**: Easy Approach to Requirements Syntax — the pattern used for all acceptance criteria in this document.

## Requirements

### Requirement 1: Fix SSE→JSON Parsing Bug in Compose Route

**User Story:** As a content operator, I want the Compose page to correctly read the streaming progress from `/api/post-video`, so that I see live progress updates instead of a JSON parse error.

#### Acceptance Criteria

1. WHEN the Composer calls `/api/post-video` and the server responds with `Content-Type: text/event-stream`, THE Composer SHALL read the response body as a `ReadableStream` using `response.body.getReader()` and NOT call `response.json()`.
2. WHEN a line in the SSE stream starts with `data: `, THE Composer SHALL parse only the substring after `data: ` as JSON.
3. IF the SSE line does not start with `data: `, THEN THE Composer SHALL skip that line without throwing an error.
4. WHEN the parsed SSE event contains `{ done: true }`, THE Composer SHALL treat the post as complete and update the UI accordingly.
5. IF `response.body` is null or the response status is not 2xx, THEN THE Composer SHALL display an error message to the operator without crashing.
6. THE Composer SHALL buffer incomplete SSE lines across `reader.read()` chunks and only parse complete `data: …\n\n` events.

### Requirement 2: Fix Dark/Blank Video Thumbnails on Instagram

**User Story:** As a content operator, I want Instagram Reels to display a branded cover image instead of a black frame, so that posts look professional and attract engagement.

#### Acceptance Criteria

1. WHEN a Reel is submitted to the Meta Graph API, THE Video_Pipeline SHALL include a `cover_url` field in the IG media container creation payload pointing to a publicly accessible JPEG URL.
2. THE Cover_Image SHALL be the branded PPP TV thumbnail generated by `generateImage()` in `image-gen.ts`, staged to R2 via the Worker's `/stage-image` endpoint before the IG API call.
3. IF the `/stage-image` Worker call fails, THEN THE Video_Pipeline SHALL attempt to upload the cover image via the Facebook Graph API `/photos` endpoint as a fallback and use the returned image URL as `cover_url`.
4. IF both cover image staging methods fail, THEN THE Video_Pipeline SHALL submit the Reel without `cover_url` rather than aborting the post.
5. THE Cover_Image SHALL have dimensions of 1080×1350 pixels (4:5 ratio) to match Instagram's recommended Reel cover format.
6. WHEN the `cover_url` is included in the IG container payload, THE Meta_Graph_API SHALL receive a URL that is publicly accessible without authentication (R2 public bucket URL).

### Requirement 3: Post-Success Navigation

**User Story:** As a content operator, I want the Compose page to navigate away after a successful post, so that I am not left on a stale form after publishing.

#### Acceptance Criteria

1. WHEN a post completes with `{ done: true, success: true }` from the SSE stream, THE Composer SHALL navigate to the Cockpit tab (or `/dashboard`) after a 3-second success display delay.
2. WHEN navigating away after success, THE Composer SHALL reset the URL input, headline, caption, thumbnail, and resolved video URL fields to their empty defaults.
3. IF the post completes with `{ done: true, success: false }`, THEN THE Composer SHALL remain on the Compose tab and display the error details without navigating away.
4. WHILE the post is in progress (`status === "posting"`), THE Composer SHALL disable the Post button and prevent duplicate submissions.

### Requirement 6: Autonomous Video Scraping — Football Platforms

**User Story:** As a content operator, I want the system to autonomously scrape video content from major football platforms, so that sports content is included in the automated posting pipeline without manual intervention.

#### Acceptance Criteria

1. THE Video_Pipeline SHALL scrape video content from the following football RSS/JSON sources: ESPN (`https://www.espn.com/espn/rss/news`), Sky Sports Football (`https://www.skysports.com/rss/12040`), Goal.com (`https://www.goal.com/feeds/en/news`), and BBC Sport (`https://feeds.bbci.co.uk/sport/rss.xml`).
2. WHEN scraping football sources, THE Video_Pipeline SHALL extract the article title, article URL, publication date, and any embedded YouTube video ID from the RSS `<description>` field.
3. IF a football RSS item contains an embedded YouTube video ID, THEN THE Video_Pipeline SHALL construct a `VideoItem` with `sourceType: "youtube"` and `url: "https://www.youtube.com/watch?v={videoId}"`.
4. THE Video_Pipeline SHALL assign `category: "SPORTS"` to all items scraped from football platform sources.
5. WHEN a football video item is selected for posting, THE Video_Pipeline SHALL use the article title as the base for AI caption generation via `generateAIContent()`.
6. THE Video_Pipeline SHALL apply the existing Bloom_Filter dedup check to football video items before adding them to the posting queue.
7. IF a football RSS feed returns a non-2xx HTTP status or times out after 10 seconds, THEN THE Video_Pipeline SHALL log a warning and continue processing remaining sources without failing the entire scrape run.

### Requirement 7: Autonomous Video Scraping — Entertainment Platforms

**User Story:** As a content operator, I want the system to autonomously scrape video content from TMZ, ET Online, and Kenyan YouTube channels, so that entertainment content is continuously available for automated posting.

#### Acceptance Criteria

1. THE Video_Pipeline SHALL scrape video content from TMZ (`https://www.tmz.com/rss.xml`) and ET Online (`https://www.etonline.com/news/rss`) RSS feeds.
2. THE Video_Pipeline SHALL scrape the YouTube RSS feeds for Citizen TV Kenya (channel ID: `UCwmZiChSZyQni_AIBiYCjaA`), KTN News Kenya (channel ID: `UCt3bgbxSBmNNkpVZTABm_Ow`), and NTV Kenya (channel ID: `UCXyLMXgT-jg3wQHkMSMqmcA`).
3. WHEN scraping entertainment RSS feeds, THE Video_Pipeline SHALL extract the title, link, publication date, and `<media:thumbnail>` or `<media:content>` image URL.
4. THE Video_Pipeline SHALL assign `category: "CELEBRITY"` to TMZ and ET Online items, and `category: "NEWS"` to Kenyan TV channel items.
5. WHEN a YouTube channel RSS item is scraped, THE Video_Pipeline SHALL construct a `VideoItem` with `sourceType: "youtube"`, `url: "https://www.youtube.com/watch?v={videoId}"`, and `thumbnail: "https://img.youtube.com/vi/{videoId}/maxresdefault.jpg"`.
6. THE Video_Pipeline SHALL apply the `isEntertainmentTitle()` filter to TMZ and ET Online items to exclude non-entertainment content.
7. THE Video_Pipeline SHALL return at most 5 items per entertainment RSS feed per scrape run to avoid overwhelming the posting queue.

### Requirement 8: Thumbnail/Cover Image Extraction from Video

**User Story:** As a content operator, I want the system to extract a real frame from a video as the cover image for Instagram posts, so that Reels display a meaningful preview instead of a black frame.

#### Acceptance Criteria

1. WHEN a video is staged in R2 and a YouTube video ID is available, THE Video_Pipeline SHALL use the YouTube thumbnail URL (`https://img.youtube.com/vi/{videoId}/maxresdefault.jpg`) as the source image for `generateImage()`.
2. WHEN a TikTok video is staged and the TikWM API response includes a `cover` or `origin_cover` URL, THE Video_Pipeline SHALL fetch that URL and use it as the source image for `generateImage()`.
3. WHEN a direct MP4 video is staged and no platform thumbnail is available, THE Video_Pipeline SHALL use the `VideoItem.thumbnail` field (if non-empty) as the source image for `generateImage()`.
4. THE Cover_Image SHALL be generated by passing the thumbnail URL as `article.imageUrl` to `generateImage()`, which overlays the PPP TV logo, category pill, and headline on the thumbnail.
5. IF no thumbnail URL is available from any source, THEN THE Video_Pipeline SHALL generate a Cover_Image using a solid black background with the PPP TV logo and headline overlay.
6. THE Cover_Image SHALL be staged to R2 via the Worker's `/stage-image` endpoint and the returned public URL SHALL be passed as `cover_url` in the IG Reels API payload.
7. FOR ALL Cover_Images generated, the resulting JPEG SHALL have dimensions of 1080×1350 pixels (4:5 ratio) as enforced by `generateImage()`.
